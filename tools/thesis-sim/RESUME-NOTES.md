## Why the simulator is now resilient

`scripts/simulate-tester-soak.mjs:authedHttp` has two recovery paths:

1. **Network blip** (`status === 0`, fetch terminated / fetch failed):
   up to 3 retries with linear backoff. Handles transient Lightsail
   connection drops under bot-scanner load.
2. **JWT cycled mid-session** (`status === 401`): first tries
   `/auth/claim` (works when the seat-sweep window has freed the
   seat); on 409 same-user falls through to `/auth/login` with the
   seeded `'devcon'` password. `/auth/login` does not check seat
   ownership, so the user picks up with a fresh JWT against the same
   user row without losing the in-flight session.

This is why the resume can safely run with the same simulator without
needing further code changes.
